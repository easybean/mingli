# Mingli Refactor Plan

最后更新：2026-06-10

## 目标

把 Mingli 从“排盘 + 资料展示 + 游戏模块”重构成一条清晰的用户路径：

`排盘 -> 今日关卡 -> 最近一月 -> 人生主线 -> 深度解读`

这次重构不以“功能更多”为目标，而以三件事为目标：

- 用户进来 3 秒内知道该做什么
- 手机端成为主体验，不再是 PC 缩小版
- 人生游戏成为可回访的主入口，而不是解读页的附属模块

## 基线

- 重构前基线提交：`90962b7 建立重构前移动端与技能基线`
- 重构前基线标签：`0.0.3`
- 本轮计划文件从 `0.0.3` 之后开始

## 工作角色

每一轮重要改动按以下角色顺序执行。

### 1. product-manager

控制项目节奏。

每轮必须先明确：

- 本轮目标
- 本轮不做什么
- 拆成几步
- 每步完成标准
- 做到什么程度停下来

### 2. product-designer

定义用户路径和页面结构。

每页必须先回答：

- 用户为什么打开这个页面
- 第一眼该看到什么
- 第一件事该做什么
- 第二步去哪里
- 哪些内容不该出现在首屏

### 3. mobile-ux

约束手机体验。

每页手机端必须满足：

- 单列为主
- 单手可点
- 导航不换行、不依赖横滑猜测
- 首屏只保留最重要路径
- 卡片按任务顺序排列，而不是按后端数据结构排列

### 4. fortune-game-designer

约束人生游戏和每日关卡。

游戏相关页面必须满足：

- 有情境
- 有选择
- 有反馈
- 有下一步
- 有回访理由
- 有分享点或结果标签

### 5. ui-reviewer

每页完成后做设计审查。

必须检查：

- 视觉层级
- 留白和密度
- 字体大小
- 按钮是否像可操作入口
- 是否仍有后台风、表单风、资料堆叠感

## 总体优先级

1. 用户路径清晰
2. 手机可用性
3. 游戏留存
4. 解读可读性
5. 分享和增长
6. 更深规则和内容扩展

如果发生冲突，先保用户路径和手机体验。

## 阶段 1：产品骨架重构

目标：统一全站主路径，避免每个 tab 各讲各的。

### 任务

- [ ] 明确首页、人生游戏、解读、命盘、时间轴的页面职责
- [ ] 确定默认用户路径：`今日 -> 近期 -> 长线 -> 解读`
- [ ] 明确每个页面的首屏主行动
- [ ] 明确哪些内容降级为二级信息或折叠信息

### 页面职责

| 页面 | 新职责 | 不再承担 |
| --- | --- | --- |
| 首页/总览 | 入口和路径分发 | 展示所有摘要 |
| 人生游戏 | 主体验和回访入口 | 静态资料列表 |
| 解读 | 深度解释和依据 | 首屏主路径 |
| 命盘 | 工具查询 | 新用户入口 |
| 时间轴 | 运限查询 | 主游戏体验 |

### 验收标准

- 用户进入首页能明确看到一个主按钮
- 首页不再像摘要仓库
- 每个 tab 的用途不重叠
- 不需要解释，用户也能知道下一步去哪

## 阶段 2：首页/总览重构

目标：首页成为真正的产品入口。

### 用户目标

用户不是来读一份大报告，而是想知道：

- 我今天该注意什么
- 最近一段时间怎么走
- 我的长期主线是什么

### 页面结构

1. 排盘输入区
2. 今日入口主卡
3. 最近一月入口
4. 人生主线入口
5. 少量身份摘要
6. 深度解读入口

### 任务

- [ ] 手机首屏优先展示输入和生成动作
- [ ] 排盘后自动聚焦到今日入口或总览主卡
- [ ] 首页 CTA 固定为三段：`今日关卡 / 最近一月 / 人生主线`
- [ ] 摘要信息压缩到 3-4 个关键标签
- [ ] 依据、书籍、命中细节不在首页首屏展示

### 不做

- 不在首页展示完整十二宫
- 不在首页展示大段命理解释
- 不在首页展示全部知识依据

### 验收标准

- 手机首屏不拥挤
- 第一主按钮明确
- 三个入口主次清楚
- 首页读起来像路径，不像资料拼盘

## 阶段 3：人生游戏重构

目标：把人生游戏做成主体验，而不是解读模块的游戏化展示。

### 用户目标

用户打开人生游戏，是为了获得一个可参与的“当前人生局”：

- 当前遇到什么题
- 我能怎么选
- 选完有什么反馈
- 下一题是什么
- 这说明我是什么路线

### 页面结构

1. 当前局势
2. 当前关卡
3. 三个选择
4. 选择反馈
5. 下一关
6. 行动日志
7. 阶段地图

### 任务

- [ ] 当前题目成为页面主视觉，不让角色资料抢位置
- [ ] 今日关卡默认只展示一题
- [ ] 选项文案改成更生活化的情境选择
- [ ] 反馈区突出“你刚才的选择改变了什么”
- [ ] 下一题入口更明显
- [ ] 行动日志做成轻量时间线
- [ ] 阶段地图降级，避免压过当前题

### 游戏循环

`情境 -> 选择 -> 代价/收益 -> 反馈 -> 路线变化 -> 下一题`

### 验收标准

- 页面像游戏流程，不像资料页
- 当前题和可选动作一眼可见
- 选择后能明显看到反馈和下一步
- 手机端一屏内能完成一次选择判断

## 阶段 4：每日关卡和近期关卡提质

目标：提升回访和留存。

### 用户目标

用户每天回来，不是为了看同一套解释，而是想知道：

- 今天有什么小提醒
- 今天更适合推进、修复还是等待
- 今天的选择会影响什么路线

### 任务

- [ ] 今日关卡文案从抽象命理词改成具体生活情境
- [ ] 最近一月拆成阶段感更强的 3-4 个小关卡
- [ ] 增加结果标签，例如：`今日稳盘 / 今日破局 / 今日修复`
- [ ] 增加可分享摘要文案
- [ ] 增加连续回访提示，但第一版不做账号和存档

### 不做

- 不做复杂数值系统
- 不做账号系统
- 不做真实付费

### 验收标准

- 今日关卡普通用户能看懂
- 每个选择有明确代价和收益
- 结果有一句适合分享的总结
- 用户能理解为什么明天还可以再来

## 阶段 5：解读页重构

目标：把解读页从资料堆叠改成分层阅读。

### 用户目标

用户从游戏或首页进入解读，是为了理解：

- 为什么我是这个路线
- 命盘结构怎么支持这个判断
- 更深依据在哪里

### 页面结构

1. 用户语言结论
2. 结构解释
3. 命理依据
4. 书籍引用
5. 技术细节

### 任务

- [ ] 首屏只保留 3-5 条核心结论
- [ ] 八字和紫微分层，不混成一大段
- [ ] 十二宫默认折叠，只突出命宫、身宫、当前相关宫位
- [ ] 知识命中、检索片段、专题依据默认折叠
- [ ] 移除或下沉 `AI Prompt Seed`

### 验收标准

- 普通用户能先读懂结论
- 懂命理的人能继续展开依据
- 手机端不再连续出现大量长文本
- 解读页不抢人生游戏的主路径

## 阶段 6：命盘页重构

目标：命盘页成为工具查询页。

### 用户目标

懂命理或想看结构的人，在这里查看：

- 十二宫
- 主星/辅星
- 四化
- 当前宫位重点

### 任务

- [ ] PC 保留完整命盘
- [ ] 手机端优先展示宫位列表
- [ ] 增加“当前重点宫位”摘要
- [ ] 十二宫卡片支持按主题筛选或快速跳转
- [ ] 命盘页不承担新用户解释任务

### 验收标准

- 手机端不再依赖横向大盘作为主要查看方式
- 用户能快速找到命宫、身宫、财帛、官禄、夫妻等重点宫位
- 完整命盘仍然保留给高级用户

## 阶段 7：时间轴/运限页重构

目标：时间轴服务于阶段理解，不抢游戏主线。

### 用户目标

用户想知道：

- 当前在哪一步大运
- 未来几年主题怎么变化
- 最近一年/月/日为什么这样

### 任务

- [ ] 当前大运放在最前
- [ ] 未来十年用阶段卡表达
- [ ] 流年/流月/流日作为二级展开
- [ ] 与人生游戏近期视角互相跳转

### 验收标准

- 当前年龄和当前大运清晰
- 不再从出生第一步大运开始压迫用户
- 用户能从时间轴跳到对应游戏视角

## 阶段 8：增长与分享

目标：在不破坏可信度的前提下增加分享和回访。

### 任务

- [ ] 增加人生原型结果卡
- [ ] 增加今日路线结果卡
- [ ] 增加最近一月主题摘要
- [ ] 增加适合截图分享的短结果
- [ ] 规划付费点，但不在第一轮实现

### 可选增长点

- 今日挑战分享
- 人生原型分享
- 当前路线分享
- 最近一月提醒
- 深度报告解锁

### 验收标准

- 分享文案不夸张、不恐吓
- 截图内容有身份感
- 用户愿意把结果发给别人看

## 执行节奏

每一页按固定节奏推进：

1. PM 定范围
2. Product Designer 定路径
3. Mobile UX 定手机结构
4. 实现代码
5. UI Reviewer 审查
6. `npm run verify`
7. 更新计划
8. 再进入下一页

## 第一轮执行顺序

1. 首页/总览
2. 人生游戏
3. 每日关卡文案
4. 解读页
5. 命盘页
6. 时间轴页
7. 分享与增长

## 第一轮不做

- 不新增更多书
- 不扩展新命理体系
- 不接 AI 生成
- 不做账号
- 不做支付
- 不做复杂存档
- 不大规模重写后端规则

## 每轮完成标准

一轮改动只有同时满足以下条件才算完成：

- 用户路径更清楚
- 手机端体验更好
- 没有新增明显信息噪音
- 样例验证通过
- 计划文件同步更新
