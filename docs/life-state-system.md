# Mingli Life State System v1

本文件定义“人生状态系统 v1”。它解决的问题是：命盘解读可以保持稳定，但用户在关卡中的选择应该改变生活状态、后续局面和最终路线总结。

## 1. 产品原则

### 命盘不变，生活状态会变

命盘是底盘：

- 决定用户容易遇到哪类关卡。
- 决定事业、关系、财务、健康、心性等题库权重。
- 决定某些题目更容易出现。

用户选择是走法：

- 不改命盘。
- 不改出生结构。
- 改变当前生活状态。
- 改变后续关卡的压力、机会、关系和资源语气。

### 游戏表达

用户不应该感觉自己只是在看固定解读，而应该感觉：

```text
同一件事，我选不同做法，后面的生活会变得不一样。
```

## 2. v1 状态字段

第一版只做 6 个生活状态，避免复杂数值系统。

| 字段 | 中文 | 含义 |
| --- | --- | --- |
| `pressure` | 压力 | 当前承担、紧绷、被推动的程度 |
| `opportunity` | 机会 | 曝光、资源、突破窗口的程度 |
| `relationship` | 关系 | 协作、信任、边界清晰度 |
| `stability` | 稳定 | 节奏、秩序、长期可持续性 |
| `resources` | 资源 | 时间、钱、人脉、可调用支持 |
| `wellbeing` | 身心 | 体力、恢复、心理余量 |

### 数值范围

```text
0 - 100
初始值：50
```

### 状态解释

| 区间 | 表达 |
| --- | --- |
| 0-29 | 紧张，需要优先处理 |
| 30-44 | 偏弱，要谨慎消耗 |
| 45-60 | 正常，可灵活选择 |
| 61-75 | 偏强，可以利用 |
| 76-100 | 很强，但可能伴随副作用 |

## 3. 选择影响规则

### 当前模板选择风格映射

现有模板里大多是：

- `bold`
- `steady`
- `repair`

v1 先兼容这三类，但不把页面永远写成主动/稳健/修复。

| style | 主要变化 | 代价 |
| --- | --- | --- |
| `bold` | 机会上升、资源可能上升、主导感上升 | 压力上升、身心下降 |
| `steady` | 稳定上升、关系小幅上升、压力下降 | 机会增长较慢 |
| `repair` | 身心上升、关系边界改善、压力下降 | 机会可能下降、推进变慢 |

### 建议默认 delta

```js
bold: {
  pressure: +8,
  opportunity: +9,
  relationship: -2,
  stability: -3,
  resources: +2,
  wellbeing: -5
}

steady: {
  pressure: -2,
  opportunity: +3,
  relationship: +3,
  stability: +8,
  resources: +3,
  wellbeing: +1
}

repair: {
  pressure: -7,
  opportunity: -2,
  relationship: +6,
  stability: +4,
  resources: -1,
  wellbeing: +8
}
```

### 主题修正

同样的选择，在不同主题里影响不同。

| theme | 修正 |
| --- | --- |
| career | `opportunity` 和 `pressure` 权重更高 |
| wealth | `resources` 和 `stability` 权重更高 |
| relationship | `relationship` 和 `pressure` 权重更高 |
| health | `wellbeing` 和 `stability` 权重更高 |
| mindset | `wellbeing` 和 `pressure` 权重更高 |
| family | `relationship` 和 `resources` 权重更高 |
| migration | `opportunity` 和 `stability` 权重更高 |

v1 不做复杂加权，只在文案解释里体现主题影响。

## 4. 反馈文案结构

选择后反馈不再只是：

```text
你选择了主动路线。
```

而是：

```text
这次选择让机会升高，但压力也被抬高。
短期你更容易被看见，后续要小心恢复不足。
```

### 反馈结构

每次选择后展示：

1. 本次路线结果。
2. 主要正向变化。
3. 主要代价。
4. 下一关风险提示。

## 5. 状态影响后续关卡

v1 先不重新生成题库，只改变展示和排序语气。

### 状态触发文案

| 条件 | 后续提示 |
| --- | --- |
| `pressure >= 70` | 后续关卡更容易出现恢复、边界、节奏问题 |
| `opportunity >= 70` | 后续关卡更容易出现曝光、承担、选择窗口 |
| `relationship <= 35` | 后续关卡更容易出现沟通、信任、边界摩擦 |
| `stability <= 35` | 后续关卡更容易出现计划打乱、反复调整 |
| `resources <= 35` | 后续关卡更容易出现取舍、预算、求助问题 |
| `wellbeing <= 35` | 后续关卡更容易出现休息、恢复、身心余量问题 |

## 6. 页面展示

### 今日关卡

选择后展示：

- 今日生活变化。
- 状态变化标签。
- 明日提示。

### 人生游戏

顶部展示简化状态：

```text
当前生活状态
压力 58 · 机会 64 · 关系 47
```

不展示 6 条大进度条，避免像后台面板。

### 游戏结束

最后一关后展示：

- 当前生活路线总结。
- 最强状态。
- 最需要照看的状态。
- 下一步建议。

## 7. 选项类型多样化衔接

人生状态系统先兼容旧 `style`。

下一阶段再把选项从固定三件套扩成事件动作，例如：

- 接下 / 拒绝 / 谈条件
- 表达 / 沉默 / 转移
- 投入 / 观望 / 止损
- 求助 / 独自处理 / 延期

这些动作仍然映射到 `lifeState` 变化，但页面不再暴露固定的主动/稳健/修复标签。

## 8. v1 不做什么

- 不做账号存档。
- 不做复杂 RPG 属性系统。
- 不让状态改变命盘解读本体。
- 不做真实医疗、投资、法律建议。
- 不做长期数据库记录。

## 9. v1 验收标准

- 每次选择后，用户能看到生活状态变化。
- 不同选择会产生不同状态结果。
- 状态变化会影响后续关卡的提示语。
- 最后一关有生活路线总结。
- 命理解读仍然稳定，不被用户选择改写。
