# Fortune Game Designer

Use this skill for the life-game and daily/near-term challenge system.

## Goal

Make the user feel:

- this is about me
- I want to keep going
- I want to come back
- I want to share this

## Core principles

- Use destiny as framing, not as rigid prediction.
- The game should feel personal, not like a generic quiz.
- A challenge should create tension, choice, and consequence.
- The next step should always be visible.

## Required output before coding

For any new game feature, first define:

1. `用户此刻为什么会打开`
2. `这一轮的核心情绪`
3. `本轮目标`
4. `一题完成后用户为什么还想继续`
5. `分享点`
6. `复测点`
7. `潜在付费点`

## Mingli game loop

Prefer this order:

1. hook: today's question
2. tension: what is the current issue
3. choice: 2-3 meaningful branches
4. feedback: what changed
5. continuation: what comes next
6. reflection: what this says about the user pattern

## Content rules

- Write clearly enough for non-metaphysics users.
- Keep metaphysics flavor, but translate it into life language.
- Avoid empty abstract phrases.
- Avoid disaster prophecy.
- Every question should feel like a lived situation, not a textbook label.

## Growth rules

- Think about return frequency: day, week, month.
- Think about shareability: archetype, route, result snapshot, “today's challenge”.
- Think about progression: current streak, current route, unresolved thread.
- Think about premium potential: deeper route unlocks, custom long-term storylines, richer guidance.

## Hard bans

- no admin-dashboard tone
- no raw data dump as gameplay
- no “choose A/B/C” without emotional framing
- no static list of all events shown upfront
