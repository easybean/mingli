# Mingli UI Design

本文件是 Mingli 重构的 UI 设计稿。它承接 `mingli-product-design-detail.md` 和 `mingli-mobile-ux-design.md`，用于指导后续前端实现。

本阶段不写代码。确认本文件后，才进入实现。

## 1. 视觉方向

### 目标气质

Mingli 的界面应该是：

- 安静。
- 有一点神秘感。
- 像一个个人命运游戏，而不是后台工具。
- 可读、克制、具体。
- 手机端像独立产品，不像 PC 页面缩小版。

### 不要的感觉

- 后台管理系统。
- 表单工具。
- 资料库。
- 玄学广告页。
- 过度装饰的国风页面。
- 所有卡片都一样重。

### 视觉关键词

```text
沉静 / 纸感 / 微光 / 选择感 / 阶段感 / 个人化
```

## 2. 色彩系统

### 主背景

用暖白或浅纸色，不用纯白。

建议：

```text
page-bg: #F8F3E8
surface: #FFFDF7
surface-soft: #F3EBDC
```

### 主色

主色用于按钮、当前导航、关键状态。

建议：

```text
primary: #245C63
primary-deep: #173F45
primary-soft: #DDECEE
```

原因：

- 比蓝紫更沉稳。
- 有东方感，但不俗套。
- 可同时承载命理和游戏气质。

### 辅助色

用于近期、机会、提醒。

```text
gold: #B98532
green: #5F7F4B
red-soft: #B96A58
ink: #2C2620
muted: #7A7065
line: rgba(44, 38, 32, 0.12)
```

### 禁止

- 大面积紫蓝渐变。
- 大面积深色背景。
- 大面积金色导致廉价玄学感。
- 每个模块都用不同高饱和色。

## 3. 字体层级

### 手机端

```text
页面大标题：24px / 1.2 / 600
页面副标题：14px / 1.5 / 400
卡片标题：18px / 1.25 / 600
关卡标题：20px / 1.25 / 650
正文：15px / 1.5 / 400
辅助说明：12px / 1.4 / 400
按钮：15px / 1.2 / 600
底部导航：11px / 1.1 / 500
```

### 文案密度

- 一张主卡最多 1 个标题、1 段正文、1 个主按钮。
- 一段正文尽量 2-4 行。
- 不在按钮里放长句。
- 状态标签最多 2-5 个字。

## 4. 间距规则

### 手机页面

```text
页面左右：16px
顶部安全区后首块：16-20px
主卡 padding：18px
普通卡 padding：14-16px
卡片间距：12px
区块间距：20px
底部导航高度：58-66px
```

### 控制原则

- 不用大面积空白撑高级感。
- 游戏页面要更紧凑。
- 解读页面可以稍微松一点。
- 同一屏优先保证“可行动”，不是“看起来内容很多”。

## 5. 卡片样式

### 主卡

用于今日关卡、当前人生关卡。

视觉：

```text
背景：surface
圆角：18px
边框：1px line
阴影：轻微，不做浮夸投影
内边距：18px
```

主卡可以有轻微顶部色带或小标签，但不使用大面积渐变。

### 次卡

用于最近一月、人生主线、身份摘要。

```text
背景：surface
圆角：14px
边框：1px line
padding：14px
```

### 选项卡

用于今日/人生游戏选择。

```text
默认：浅纸色背景 + 边框
选中：primary-soft 背景 + primary 边框
弱化：透明度降低，但文字仍可读
```

选项卡整卡可点。

## 6. 按钮样式

### 主按钮

用于生成、开始今日关卡、继续下一关。

```text
高度：48px
圆角：14px
背景：primary
文字：白色
字号：15px
字重：600
```

### 次按钮

用于最近一月、人生主线、深度解读。

```text
高度：42px
圆角：12px
背景：primary-soft 或 surface
边框：1px line
文字：primary-deep
```

### 文本按钮

用于“查看依据”“展开行动日志”。

```text
无背景
颜色：primary
字号：13px
```

## 7. 底部导航 UI

### 结构

```text
今日 | 游戏 | 解读 | 命盘 | 我的
```

### 样式

```text
高度：60px
背景：rgba(255, 253, 247, 0.94)
顶部边框：1px line
当前项：primary 色 + 小圆点/短线
未选项：muted
```

### 图标

如果实现阶段使用 icon，建议：

- 今日：sun
- 游戏：sparkles 或 route
- 解读：book-open
- 命盘：circle-dot
- 我的：user

图标要小，不抢文字。

## 8. 手机首页高保真文字稿

### 未生成状态

```text
Mingli

开启你的今日人生关卡
用出生信息生成命盘，把今天、近期和人生主线变成可以选择的关卡。

[出生信息]
性别        男 / 女
出生日期    1992-08-18
出生时间    08:30
出生地点    江苏 徐州
当前时间    2026-06-10

[生成我的关卡]

这是倾向与选择模拟，不是绝对预言。
```

### 已生成状态

```text
今天的关卡

一个临时变化可能打乱你的节奏。
你可以选择主动接住，也可以先把边界说清楚。

主题
工作 · 节奏 · 边界

[开始今日选择]

[最近一月]
看眼前节奏

[人生主线]
看长期路线

身份摘要
破局者 · 当前大运 · 今日关键词

想看为什么？进入深度解读
```

### 首页视觉重点

- “今天的关卡”是最大视觉块。
- 最近一月和人生主线是并列次入口。
- 身份摘要只是小标签，不做大解释。
- 深度解读是低调入口。

## 9. 今日关卡高保真文字稿

### 选择前

```text
今日关卡
第 1 题

今天有人把一个临时任务推给你。
接下它可能让你被看见，但也会打乱原计划。

你怎么处理？

[A]
直接接下，并争取把成果展示出来
代价：压力会上升
收益：更容易被看见

[B]
只接自己能负责的部分
代价：推进会慢一点
收益：风险更可控

[C]
先处理手头计划，晚点再回复
代价：机会可能变弱
收益：节奏更稳
```

### 选择后

```text
你选择了主动路线

今天适合把“被看见”当成收益，但要提前留出恢复时间。
如果继续硬撑，后面容易变成被事情推着走。

今日变化
主导力 +8
稳定性 -3
恢复力 +2

[看最近一月]
[进入人生主线]
明天再来
```

### 今日关卡视觉重点

- 情境卡最重要。
- 选项卡要清楚区别，但不要花哨。
- 反馈卡要像“回应用户选择”，不是机械结果。
- 今日页面不展示大段依据。

## 10. 人生游戏高保真文字稿

### 当前关卡

```text
破局者路线
第 3 关 / 12 关

当前倾向
主动 42 · 稳定 31 · 修复 27

关卡：主动接旗

一个更大的职责正在靠近你。
它能让你更快被看见，但也会让你承担更多评价和压力。

命中线索
事业主线、官禄、化权。

你怎么走？

[A 主动路线]
接下更大的职责，用行动争取位置。
代价：短期压力和曝光度会上升。
收益：有机会快速打开局面。

[B 稳健路线]
先把当前角色做稳，再逐步扩大影响。
代价：突破感不强。
收益：更容易积累可信度和长期位置。

[C 修复路线]
明确哪些责任该接，哪些责任需要拒绝。
代价：可能经历一次关系或职权摩擦。
收益：减少被过度消耗的概率。
```

### 选择后

```text
路线发生变化

你选择了稳健路线。
下一关会更强调积累、信任和长期位置，而不是立刻突破。

本次影响
稳定 +7
主导 +3

[继续下一关]

行动日志
已完成 3 次选择
当前路线：稳健推进

阶段地图
当前阶段：积累可信度
下一阶段：扩大边界
```

### 人生游戏视觉重点

- 顶部只展示路线、进度和当前倾向。
- 当前关卡占主视觉。
- 选择卡可以较长，但每张不能超过 5-6 行。
- 行动日志和阶段地图默认折叠。

## 11. 解读页 UI 方向

### 首屏

```text
深度解读
这里解释为什么系统给出这些关卡。

[目录]
八字基础
紫微十二宫
格局
事业
财运
婚恋
健康
心性
依据
```

### 内容结构

每个专题：

```text
结论
一句话说明

结构原因
2-4 条

展开依据
书籍 / 知识命中 / 检索片段
```

### 视觉规则

- 默认展开最重要的 1-2 个专题。
- 其他专题折叠。
- 依据折叠到最深层。
- 不展示 AI Prompt Seed 给普通用户。

## 12. 我的页 UI 方向

### 页面职责

承载不适合放在主路径里的信息。

### 内容

```text
我的命盘
出生信息
当前定位时间
当前路线
今日结果

设置
重新生成
说明
免责声明
```

### 作用

避免首页堆太多设置和说明。

## 13. 实现前验收清单

进入代码实现前，需要确认：

- 手机端是否采用底部导航。
- 首页是否以今日关卡为主入口。
- 今日关卡是否只做一题。
- 人生游戏是否默认只显示当前关卡。
- 深度解读是否降级为解释层。
- 是否接受这套色彩方向。
- 是否接受整体舍弃当前 index 布局风格。

确认后才进入前端实现。
