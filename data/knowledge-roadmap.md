# Mingli Knowledge Roadmap

## Goal

把 `data/knowledge/` 里的紫微与八字书籍系统化接入项目，形成四层能力：

1. `rules`: 结构化规则命中层，直接参与后端判断。
2. `references`: 引用依据层，为结论提供出处与旁证。
3. `topics`: 专题解读层，按婚恋、事业、财运等主题组织知识。
4. `retrieval`: 轻量检索层，按命盘关键词提取少量文本片段增强 `aiPromptSeed`。

当前项目只处理 `ziwei` 与 `bazi`，其他体系维持 `defer`。

## Delivery Phases

### Phase 1: Foundation Expansion

目标：把现有“只接两本规则源”的结构扩成“可持续接书”的框架。

交付件：

- 扩充 `data/knowledge-meta.json`
- 新增批次字段、专题字段、优先级字段
- 规范规则文件分层命名
- 规范引用片段和专题片段的数据结构

### Phase 2: Rules Layer

目标：把高价值的紫微/八字书籍抽成可命中的结构化规则。

优先处理：

- 八字：
  - `《子平真诠》`
  - `《滴天髓阐微》`
  - `《穷通宝鉴》`
  - `《三命通会》`
- 紫微：
  - `《斗数发微论》`
  - `《紫微斗数一本通》`
  - `《斗数骨髓赋》`
  - `《增补太微赋》`

规则方向：

- 八字：
  - 月令
  - 格局
  - 用神/忌神
  - 日主旺衰
  - 十神组合
  - 财官印食伤专题判断
- 紫微：
  - 主星在宫
  - 宫位组合
  - 身命联动
  - 四化落宫
  - 事业/婚恋/财帛等主线判断

### Phase 3: Reference Layer

目标：让主要解读都能挂到出处。

优先处理：

- 八字引用：
  - `《渊海子平》`
  - `《千里命稿》`
  - `《神峰通考》`
  - `《命理探原》`
  - `《玉照定真经》`
- 紫微引用：
  - `《斗数玄微论》`
  - `《重补斗数彀率》`
  - `《女命骨髓赋》`
  - `《斗数十喻歌》`
  - `《斗数观音经验谈》`

输出方式：

- `reading.references`
- `reading.manual[].references`
- `reading.knowledgeHits[].source`

### Phase 4: Topic Layer

目标：把书籍内容按用户更容易理解的主题收束，而不是只按术数概念输出。

首批专题：

- `career`
- `wealth`
- `marriage`
- `health`
- `mindset`

每个专题都允许同时吸收 `ziwei` 与 `bazi` 的规则和引用，但必须分开标注来源体系。

### Phase 5: Retrieval Layer

目标：在不污染规则判断的前提下，为 `aiPromptSeed` 注入少量相关书籍片段。

约束：

- 只检索 `ziwei` 与 `bazi`
- 先由命盘生成关键词，再检索文本片段
- 每次只取极少量高相关片段
- 检索结果只做增强，不直接决定结论

## Engineering Sequence

1. 扩充元数据结构，标出全部紫微/八字书籍的批次与专题。
2. 做 `stage2` 规则文件，先补八字格局和紫微主星落宫。
3. 改 `knowledge.js`，支持多批次规则聚合。
4. 做引用片段索引，让更多书源进入 `references`。
5. 做专题片段，让 `manual` 从“盘面章节”升级为“盘面章节 + 主题章节”。
6. 做轻量检索，把书籍片段接入 `aiPromptSeed`。
7. 前端增加来源与专题展示，最后统一验收。

## Acceptance Standard

- 每个主要结论都能说明来自哪一层：
  - 基础盘面
  - 规则命中
  - 引用依据
  - 专题补充
  - 检索增强
- 紫微与八字不会混写成一条模糊判断。
- 新增书籍时，不需要改动主流程结构，只需要补元数据与知识文件。
