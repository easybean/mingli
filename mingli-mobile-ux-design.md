# Mingli Mobile UX Design

本文件是 Mingli 重构的移动端 UX 设计稿。后续实现时，必须以本文件为手机端基准，不再沿用现有 `public/index.html` 的布局风格做简单适配。

## 1. 重做边界

### 明确舍弃

移动端不继续沿用当前 index 的这些特征：

- 大块表单 + 大块结果平铺。
- 多个同级卡片平均展示。
- PC 信息流压缩到手机。
- 大量边框卡片连续堆叠。
- 首页同时承载总览、解读、游戏、时间轴入口。
- 生成后仍然让用户自己寻找下一步。

### 保留能力

保留底层能力，但重组入口：

- 出生信息输入。
- 排盘生成。
- 今日关卡。
- 近期关卡。
- 人生游戏。
- 深度解读。
- 命盘和时间轴。

### 手机端设计目标

用户在手机上打开后，3 秒内应知道：

1. 这里可以生成我的命盘关卡。
2. 生成后第一步是做今日关卡。
3. 想深入再去解读，不需要一开始看完全部资料。

## 2. 移动端产品结构

### 一级结构

手机端推荐采用“主路径 + 底部导航”的结构。

```text
主路径：
首页 -> 今日关卡 -> 选择反馈 -> 近期/人生主线 -> 深度解读

底部导航：
今日 / 游戏 / 解读 / 命盘 / 我的
```

### 为什么不用顶部五个 Tab

当前顶部 tab 在手机端容易出现：

- 换行。
- 字太小。
- 用户不知道哪个是主入口。
- 页面顶部空间被导航占掉。

移动端更适合底部导航：

- 单手可点。
- 一级入口更稳定。
- 顶部区域留给当前任务。

### 底部导航定义

| 导航 | 用户意图 | 内容 |
| --- | --- | --- |
| 今日 | 我今天该怎么做 | 今日关卡、今日反馈、明日提示 |
| 游戏 | 我的人生主线是什么 | 人生游戏、阶段关卡、行动日志 |
| 解读 | 为什么这么说 | 八字、紫微、格局、专题依据 |
| 命盘 | 我的盘面结构 | 十二宫、星曜、四化、时间轴入口 |
| 我的 | 我的信息和记录 | 出生信息、当前路线、设置、说明 |

### 首次用户默认页

未生成命盘时：默认进入首页输入状态。

已生成命盘时：默认进入“今日”页。

## 3. 手机首页 UX

### 未生成状态

#### 页面目标

让用户快速输入出生信息并生成关卡。

#### 文字线框

```text
[顶部]
Mingli
开启你的今日人生关卡
用出生信息生成命盘，把今天、近期和人生主线变成可以选择的关卡。

[输入卡片]
性别
出生日期
出生时间
出生地点
当前定位时间

[主按钮]
生成我的关卡

[轻说明]
这是倾向与选择模拟，不是绝对预言。
```

#### 输入区规则

- 表单不要像后台录入。
- 每行只保留必要标签。
- 日期、时间、地点优先使用原生控件。
- 高级设置默认折叠。
- 当前定位时间默认今天，不让用户一开始理解复杂概念。

### 已生成状态

#### 页面目标

把用户推向今日关卡。

#### 文字线框

```text
[今日主卡]
今天的关卡
一句生活化情境钩子
主题：工作 / 关系 / 财务 / 身心 / 成长

[主按钮]
开始今日选择

[次入口]
最近一月
看眼前节奏

人生主线
看长期路线

[身份摘要]
人生原型
当前阶段
今日关键词

[深度入口]
想看为什么？进入深度解读
```

#### 已生成状态首屏优先级

1. 今日关卡标题。
2. 今日情境一句话。
3. 开始按钮。
4. 最近一月 / 人生主线。
5. 身份摘要。

不放：

- 完整大运。
- 完整十二宫。
- 知识来源。
- 书籍依据。
- 长篇解释。

## 4. 今日关卡 Mobile UX

### 页面目标

让用户在 30 秒内完成今日一题。

### 文字线框

```text
[顶部状态]
今日关卡
第 1 题 / 今日 1 题

[情境卡]
今天有人把一个临时任务推给你。
接下它可能让你被看见，但也会打乱原计划。

[选择区]
A 直接接下，并争取展示成果
代价：压力上升
收益：更容易被看见

B 只接自己能负责的部分
代价：推进变慢
收益：风险更可控

C 先处理手头计划，晚点回复
代价：机会可能变弱
收益：节奏更稳

[选择后反馈]
你选择了主动路线。
今天适合把“被看见”当成收益，但要提前留出恢复时间。

[下一步]
看最近一月
进入人生主线
明天再来
```

### 交互规则

- 选择前只显示题目和 3 个选项。
- 选择后显示反馈，选项可保持但弱化。
- 今日关卡不展示行动日志，避免太重。
- 今日只做 1 题，保持轻。
- 如果用户想继续，进入“最近一月”或“人生游戏”。

### 选项布局

手机端选项不强求三列。

原因：

- 三列会导致文字过窄。
- 今日选择需要读懂代价和收益。
- 单列卡片更适合拇指点击。

推荐：

```text
选项卡 A
选项卡 B
选项卡 C
```

每张选项卡高度控制在 88-116px。

## 5. 人生游戏 Mobile UX

### 页面目标

让用户感觉自己在推进一条个人路线，而不是阅读列表。

### 文字线框

```text
[路线状态]
破局者路线
第 3 关 / 12 关
主动 42 · 稳定 31 · 修复 27

[当前关卡]
标题
一段 2-4 行情境
命中线索：简短一句

[选择]
A 主动路线
B 稳健路线
C 修复路线

[反馈]
你的选择让路线偏向主动。
下一关会更强调承担和曝光。

[下一关按钮]
继续下一关

[折叠区]
行动日志
阶段地图
```

### 信息顺序

1. 当前路线。
2. 当前题。
3. 选择。
4. 反馈。
5. 下一题。
6. 日志与阶段。

### 不默认展示

- 全部试炼卡。
- 全部机会卡。
- 完整阶段关卡列表。
- 所有选择结果。

这些可以作为折叠内容或完成后复盘内容。

### 路线状态展示

不要用 5 个大进度条占首屏。

推荐简化为：

```text
当前倾向：主动路线
主动 42 · 稳定 31 · 修复 27
```

五属性可以进入“我的”或折叠面板。

## 6. 近期关卡 Mobile UX

### 页面目标

让用户理解当前时间段，而不是浏览大运表。

### 入口

近期关卡从两个地方进入：

- 今日反馈后。
- 首页次入口。

### 结构

```text
[近期总览]
你现在处在：当前大运 / 当前流年

[最近一月]
本月主题
3 个小关卡

[最近一年]
年度主线
机会提醒
风险提醒

[最近十年]
从当前年龄开始
3-4 个阶段块
```

### 最近十年规则

- 必须从当前年龄开始。
- 儿童/青少年用户不写成人事业压力。
- 成年用户可以写事业、关系、财务、迁移。
- 老年用户要更多写健康、家庭、传承、生活节奏。

## 7. 深度解读 Mobile UX

### 页面目标

给想深入的人看依据，但保持可读。

### 结构

```text
[解读目录]
八字基础
紫微十二宫
格局
事业
财运
婚恋
健康
心性
依据

[内容]
默认展开最相关 1-2 项
其他折叠
```

### 规则

- 不在顶部直接铺全部内容。
- 每个专题先给结论，再给原因，再给依据。
- 书籍和检索片段默认折叠。
- AI Prompt Seed 不直接对普通用户展示，可以放开发/调试区。

## 8. 命盘与时间 Mobile UX

### 命盘页

给专业用户，不做主路径。

结构：

```text
[基础信息]
[十二宫网格/列表切换]
[格局标签]
[四化]
```

手机端十二宫可以默认列表，不强求复杂网格。

### 时间页

结构：

```text
[当前定位]
[今日]
[本月]
[今年]
[当前大运]
[未来阶段]
```

时间页应服务近期关卡，不直接制造焦虑。

## 9. 移动端视觉密度规则

### 字号

- 页面标题：22-26px。
- 卡片标题：17-20px。
- 正文：14-16px。
- 辅助说明：12-13px。
- 导航：11-12px。

### 行高

- 标题：1.2-1.3。
- 正文：1.45-1.6。
- 游戏题：1.45-1.55。
- 辅助说明：1.35-1.45。

### 间距

- 页面左右 padding：16px。
- 卡片 padding：14-18px。
- 卡片间距：10-14px。
- 大区块间距：18-24px。

### 卡片

- 卡片不要层层嵌套。
- 一屏不要同时出现太多同权重卡片。
- 主卡可以更强，次卡要更轻。
- 不要所有内容都用厚边框包起来。

### 按钮

- 主按钮高度：44-52px。
- 次按钮高度：38-44px。
- 今日关卡主按钮必须明显。
- 选项卡可以整卡点击。

## 10. 底部导航方案

### 推荐

```text
今日 | 游戏 | 解读 | 命盘 | 我的
```

### 未生成状态

底部导航可隐藏或弱化，只保留生成路径。

### 已生成状态

底部导航固定在底部。

### 导航规则

- 标签最多 2 个字。
- 不换行。
- 不横向滚动。
- 当前项要明显。
- 不使用复杂描述。

## 11. 关键移动端验收

### 首页

- 首屏能看到生成按钮或今日关卡按钮。
- 不出现大段资料。
- 不出现多个同权重入口抢焦点。

### 今日关卡

- 10 秒内读懂题目。
- 3 个选项都是具体动作。
- 选择后立刻有反馈。
- 页面不需要滚很久才看到按钮。

### 人生游戏

- 默认只显示当前关卡。
- 用户知道自己在第几关。
- 下一题按钮明显。
- 日志和阶段地图默认不抢屏幕。

### 解读

- 有目录。
- 内容折叠。
- 首屏不是术语墙。

### 全局

- 导航不换行。
- 文案不溢出。
- 卡片不显得像 PC 页面缩小版。
- 页面有主次，不是资料仓库。

## 12. 下一步：UI 设计

Mobile UX 确认后，下一步进入 UI 设计。

UI 设计需要输出：

1. 视觉方向。
2. 色彩系统。
3. 字体层级。
4. 卡片样式。
5. 按钮样式。
6. 手机首页高保真文字稿。
7. 今日关卡高保真文字稿。
8. 人生游戏高保真文字稿。

确认 UI 设计后，才进入代码实现。
