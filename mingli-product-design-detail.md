# Mingli Detailed Product Design

本文件是 Mingli 重构的产品设计稿。它先定义产品路径和页面职责，不包含具体代码实现。

## 1. 产品定位

### 一句话定位

Mingli 是一个基于命盘的人生选择游戏：用户通过出生信息生成个人命盘，再用“今日关卡、近期路线、人生主线”理解自己当前会遇到的选择，并获得可执行的行动反馈。

### 不是

- 不是传统排盘工具。
- 不是命理资料库。
- 不是把八字、紫微、书籍依据全部展示给用户的知识系统。
- 不是恐吓式预测产品。

### 是

- 一个“先玩起来，再深入看依据”的命理游戏产品。
- 一个把命理结构翻译成生活选择的体验。
- 一个让用户每天都可以回来做一题的轻量工具。

## 2. 目标用户

### 核心用户

对命理有兴趣，但不一定懂专业术语的人。

他们的真实需求通常不是“我要看完整命理体系”，而是：

- 最近我该注意什么？
- 这段时间为什么总卡住？
- 我适合主动冲，还是先稳住？
- 感情、工作、钱、健康哪个更该先处理？
- 我的长期人生主线是什么？

### 次级用户

已经懂一点八字或紫微的人。

他们会需要：

- 盘面依据。
- 宫位、十神、格局、流年大运解释。
- 书籍或知识来源。

这类内容应该存在，但不应该作为新用户首屏。

## 3. 用户心理路径

用户第一次进入时，不会先关心系统有多少书、多少规则。他的心理路径更接近：

```text
这个准不准？
-> 这个是不是在说我？
-> 今天我能做什么？
-> 我选了会怎样？
-> 后面还有什么？
-> 为什么这么说？
```

所以产品路径应当反过来设计：

```text
先给体验
-> 再给反馈
-> 再给主线
-> 最后给依据
```

## 4. 核心产品路径

### 新用户首轮路径

```text
打开页面
-> 输入出生信息
-> 点击生成
-> 看到今日关卡
-> 做一个选择
-> 看到反馈
-> 选择继续最近一月 / 人生主线 / 深度解读
```

### 老用户回访路径

```text
打开页面
-> 直接看今日关卡
-> 查看当前路线变化
-> 做一个选择
-> 查看今日反馈
-> 可选进入近期路线
```

### 深度用户路径

```text
完成今日或人生游戏
-> 对结果产生好奇
-> 进入深度解读
-> 查看八字、紫微、格局、宫位依据
-> 查看时间轴和命盘
```

## 5. 信息层级

### 第一层：行动

用户最先看到：

- 今天有什么关卡？
- 我现在该选什么？
- 选完有什么反馈？

### 第二层：趋势

用户完成第一步后看到：

- 最近一月是什么节奏？
- 最近一年有什么主题？
- 最近十年处在哪个阶段？

### 第三层：主线

用户愿意继续时看到：

- 人生主线。
- 核心试炼。
- 机会事件。
- 阶段关卡。

### 第四层：依据

用户主动深入时看到：

- 八字基础。
- 紫微十二宫。
- 格局分析。
- 知识命中。
- 书籍来源。

依据层必须作为“解释”，不能抢走产品入口。

## 6. 页面职责

### 首页 / 总览

#### 用户问题

我现在该做什么？

#### 页面目标

把用户引导到最合适的第一步。

#### 主动作

生成命盘，或开始今日关卡。

#### 内容结构

1. 出生信息输入。
2. 生成按钮。
3. 今日关卡主入口。
4. 最近一月入口。
5. 人生主线入口。
6. 简短身份摘要。
7. 深度解读入口。

#### 不展示

- 大段命理解释。
- 书籍来源。
- 知识命中。
- 所有宫位。
- 所有阶段关卡。

### 人生游戏页

#### 用户问题

我的人生会遇到什么选择？我怎么走？

#### 页面目标

让用户通过一道一道题，体验自己的命盘主线。

#### 主动作

完成当前题目。

#### 内容结构

1. 当前人生路线。
2. 当前关卡。
3. 三个选择。
4. 选择反馈。
5. 下一关。
6. 行动日志。
7. 阶段总览。

#### 关键原则

- 不默认列出全部题目。
- 一次只推一题。
- 选择后必须有反馈。
- 选择会影响后续路线文案。
- 阶段关卡只做背景，不抢当前题。

### 今日关卡页 / 今日模块

#### 用户问题

今天我最该注意什么？

#### 页面目标

做成用户每天愿意打开的最短路径。

#### 主动作

完成今日一题。

#### 内容结构

1. 今日一句话钩子。
2. 今天的具体生活情境。
3. 三个选择。
4. 选择反馈。
5. 今日路线标签。
6. 明日提示。

#### 文案原则

今日关卡必须生活化，避免只写抽象主题。

不写：

- “事业压力测试”
- “关系边界课题”
- “资源整合阶段”

改写为：

- “今天有人临时把任务推给你，你要不要接？”
- “一个表现机会突然出现，但会打乱你的计划。”
- “对方又在试探你的底线，你这次怎么回应？”

### 近期关卡页 / 近期模块

#### 用户问题

最近一个月、一年、十年我该怎么安排？

#### 页面目标

把长期命理时间变成用户能理解的节奏地图。

#### 内容结构

1. 最近一月：短期行动提醒。
2. 最近一年：年度主题。
3. 最近十年：当前大运和未来十年的阶段。

#### 关键原则

- 根据当前日期和出生日期计算当前年龄。
- 最近十年必须从当前年龄附近开始，而不是从第一个大运开始。
- 不做具体灾难预测。
- 用“阶段课题”和“推荐策略”表达。

### 深度解读页

#### 用户问题

为什么系统这么说？

#### 页面目标

承接用户的好奇心，解释命盘结构。

#### 内容结构

1. 八字基础。
2. 紫微十二宫。
3. 格局分析。
4. 事业、财运、婚恋、健康、心性专题。
5. 知识命中。
6. 书籍依据。

#### 关键原则

- 深度解读可以信息丰富。
- 但必须有目录、折叠、层级。
- 不要让新用户第一眼看到大量术语。

### 命盘页

#### 用户问题

我的盘具体长什么样？

#### 页面目标

给懂一点命理的用户查看结构化盘面。

#### 内容结构

1. 基础信息。
2. 十二宫。
3. 星曜。
4. 四化。
5. 格局标签。

#### 关键原则

命盘页是工具页，不是主路径页。

### 时间轴页

#### 用户问题

我的运势阶段怎么变化？

#### 页面目标

展示大运、流年、流月、今日之间的关系。

#### 内容结构

1. 当前时间定位。
2. 当前大运。
3. 当前流年。
4. 当前流月。
5. 今日。
6. 未来阶段。

#### 关键原则

时间轴服务近期关卡和人生主线，不直接制造焦虑。

## 7. 首页详细产品设计

### 未生成状态

#### 首屏目标

让用户知道“输入出生信息，就可以开始今日人生关卡”。

#### 首屏内容

1. 产品标题。
2. 一句解释。
3. 出生信息输入。
4. 生成按钮。

#### 首屏文案方向

标题：

```text
开启你的今日人生关卡
```

副文案：

```text
用出生信息生成命盘，把今天、近期和人生主线变成可以选择的关卡。
```

按钮：

```text
生成我的关卡
```

### 已生成状态

#### 首屏目标

让用户优先进入今日关卡。

#### 主卡结构

```text
今日关卡
一句生活化钩子
当前主题标签
开始按钮
```

#### 次级入口

```text
最近一月
看眼前节奏

人生主线
看长期关卡
```

#### 身份摘要

只保留 3-4 个短标签：

- 人生原型。
- 当前阶段。
- 主要主题。
- 今日关键词。

不要在首页解释每个标签的来源。

## 8. 人生游戏详细产品设计

### 游戏目标

用户不是来看一份报告，而是体验“我的人生路线上会出现哪些选择”。

### 当前问题

如果把所有题目、机会、试炼都列出来，用户会觉得像解读报告，不像游戏。

### 新结构

```text
路线名
当前第 N 关
情境
三个选择
反馈
下一关
行动日志
```

### 关卡组成

每道题必须有：

- 情境：一个生活场景。
- 冲突：为什么需要选择。
- 三个选择：主动、稳健、修复。
- 代价：这个选择会损耗什么。
- 收益：这个选择会带来什么。
- 反馈：系统如何回应用户。
- 后续影响：下一题的语气或路线变化。

### 路线设计

用户选择不一定要真的分出复杂树状剧情，但必须让用户感觉后续受到影响。

第一版可以用“路线状态”实现：

- 主动值高：后续文案更偏突破、承担、曝光。
- 稳定值高：后续文案更偏积累、守成、长期。
- 修复值高：后续文案更偏边界、恢复、关系整理。

### 结果设计

每完成 3-5 题，给一个小总结：

- 当前路线名称。
- 你倾向的处理方式。
- 最容易卡住的点。
- 下一阶段建议。

## 9. 今日关卡详细产品设计

### 今日关卡的产品价值

今日关卡是留存入口。它必须短、具体、有反馈。

### 今日关卡节奏

```text
今天的题
-> 10 秒读懂
-> 选一个
-> 看到反馈
-> 得到今日路线
-> 明天还想回来
```

### 今日关卡题型

#### 工作型

```text
今天有人把一个临时任务推给你。接下它可能让你被看见，但也会打乱原计划。
```

#### 关系型

```text
有人用半开玩笑的方式试探你的底线。你能感觉到不舒服，但直接说破又怕尴尬。
```

#### 财务型

```text
你看到一个想立刻下单的东西，它能让你当下舒服一点，但也会打乱预算。
```

#### 身心型

```text
你今天其实已经有点累了，但还有一件事看起来必须马上处理。
```

#### 成长型

```text
你有一个想法可以说出来，但说出来就要承担被评价的压力。
```

### 选择文案原则

每个选项都必须是动作，不是抽象态度。

不写：

```text
主动突破
稳健推进
修复关系
```

改写为：

```text
直接接下任务，并争取把成果展示出来
先确认边界，只接自己能负责的部分
先处理手头计划，晚点再回复对方
```

## 10. 近期关卡详细产品设计

### 最近一月

用途：短期行动建议。

用户看到：

- 本月主题。
- 3 个小关卡。
- 每个小关卡一个具体建议。

### 最近一年

用途：年度节奏。

用户看到：

- 年度主线。
- 上半年/下半年侧重点。
- 风险提醒。
- 机会提醒。

### 最近十年

用途：当前大运和未来十年的阶段地图。

用户看到：

- 当前年龄。
- 当前大运。
- 从当前年龄开始的 10 年阶段。
- 每 2-3 年一个阶段主题。

关键要求：

- 不能再从第一个大运开始。
- 儿童阶段不能写成年事业压力。
- 低年龄用户要转译成学习、家庭、适应、习惯、兴趣发展。

## 11. 内容语气

### 总体语气

- 神秘但不吓人。
- 具体但不绝对。
- 有命理味，但普通人能读懂。
- 像建议，不像判决。

### 禁止表达

- “一定会发生”
- “必有大灾”
- “注定失败”
- “投资必赚”
- “身体必有疾病”

### 推荐表达

- “更容易遇到”
- “可能反复出现”
- “适合提前处理”
- “这个阶段更值得关注”
- “如果选择主动推进，代价是……”

## 12. 产品验收标准

### 首页

- 用户 3 秒内知道可以生成关卡。
- 生成后第一入口是今日关卡。
- 首页不再像资料拼盘。
- 深度解读不抢主入口。

### 人生游戏

- 默认只显示当前题。
- 选择后有反馈。
- 后续题目受路线状态影响。
- 行动日志是辅助，不重复上一手。

### 今日关卡

- 题目生活化。
- 选项是具体动作。
- 反馈像对用户选择的回应。
- 有明天再来的理由。

### 手机端

- 首屏不拥挤。
- 导航不换行。
- 主按钮明显。
- 卡片短而清晰。
- 正文行间距不浪费空间。

## 13. 下一步：Mobile UX 设计

产品设计确认后，下一步才进入 Mobile UX。

Mobile UX 需要输出：

1. 手机首页文字线框。
2. 手机人生游戏文字线框。
3. 手机今日关卡文字线框。
4. 导航方案。
5. 卡片密度和按钮布局规则。

这些确认后，才进入 UI 视觉设计和代码实现。
