# Mingli 设计提示词

> 喂给 Claude Design 用的产品设计 brief。复制本文件全文作为提示词，请它给出视觉方向、页面稿和组件样式。

---

## 1. 项目介绍

Mingli 是一款把紫微斗数和八字结合的**命理产品原型**，但它的核心**不是排盘工具**。

**一句话定位：** 用出生信息生成命盘，把"今天该怎么做、最近一段路怎么走、人生主线是什么"变成可以交互的选择关卡。

**核心用户路径：**

```
排盘  →  今日关卡  →  最近一月 / 十年 / 一生主线  →  深度解读
```

**它不是：**
- 不是传统排盘工具或资料库
- 不是把八字、紫微、书籍依据全部摊给用户的知识系统
- 不是恐吓式或绝对化预测产品
- 不是 PC 后台工具

**它是：**
- 一个"先玩起来，再深入看依据"的命理游戏
- 一个把命理结构翻译成生活选择的体验
- 一个用户每天都可以回来做一题的轻量工具
- 一个手机端为主、未来要做成微信小程序的轻产品

**当前阶段：** 纯前端打磨，先把手机端 UI / UX / 文案做到位，再接后台和小程序。本轮设计**只服务手机端**，桌面端是可选自适应。

## 2. 目标用户

### 核心用户

对命理有兴趣、但不一定懂术语的成年人。

他们打开产品时的真实心理路径是：

```
这个准不准？
  → 这个是不是在说我？
  → 今天我能做什么？
  → 我选了会怎样？
  → 后面还有什么？
  → 为什么这么说？
```

所以体验顺序必须**反过来**做：先给体验，再给反馈，再给主线，最后给依据。

他们的真实问题不是"我要看完整命理体系"，而是：

- 最近我该注意什么？
- 这段时间为什么总卡住？
- 我适合主动冲，还是先稳住？
- 感情、工作、钱、健康哪个该先处理？
- 我的长期人生主线是什么？

### 次级用户

懂一点八字或紫微的人。他们要看十二宫、十神、格局、流年大运依据和书源——这些内容**存在但折叠**，不抢新用户首屏。

### 使用场景

- 手机为主，单手操作，碎片时间打开
- 通勤、午休、睡前的 1-3 分钟入口
- 不期望长时间停留，但希望明天还想再来

## 3. 主要页面列表

底部固定 5 项导航：**今日 / 游戏 / 解读 / 命盘 / 我的**。

### 3.1 输入态首页（未生成命盘）

新用户进来第一眼。

- 一句话标题：开启你的今日人生关卡
- 一句话解释：用出生信息生成命盘，把今天、近期和人生主线变成可以选择的关卡
- 出生信息卡：性别 / 出生日期 / 出生时间 / 出生地 / 当前时间（默认今天）
- 高级设置（真太阳时等）默认折叠
- 主按钮：生成我的关卡
- 底部小字：这是倾向与选择模拟，不是绝对预言

### 3.2 已生成态首页

主目标：把用户推向今日关卡。

- **今日入口主卡**（最大视觉块）：今日主题 kicker + 题目标题 + 一段生活化情境 + 主按钮"开始今日选择"
- **次入口两列并排**：最近一月 / 人生主线
- **身份摘要**（小标签，不解释）：人生原型 + 当前年龄 + 当前主题
- **深度解读入口**（低调文本按钮）：想看为什么？进入深度解读

### 3.3 今日关卡页（每天一题）

主目标：30 秒内完成今日一题。

**选择前：**
- 顶部一行：日期 + 当前主题 kicker，题目标题，右上一个"?"帮助
- 情境卡：一段 2-4 行生活场景叙述（如"上司把别人做砸的项目转交给你全权收尾"），可选一段冲突描述
- 焦点选择条（独立小条）：事业 / 财富 / 关系 / 健康 / 心态 切换，下面挂一行"如果命盘当天没有这个主题，会自动 fallback 到更强信号"的小提示
- 选择区：「你打算怎么走？」+ 3 个选项卡（每张：动作标题 + 一段如何做 + 底部"代价 / 收益"两栏）

**选择后：**
- 上方眉签 + 标题："这一手定下了"+"你选了：xxx"
- 一段 feedback（讲这次选择改变了什么，不要贴"主动路线"那种标签）
- lifeState 变化标签（如 "压力 +7、机会 +6、身心 -6"）
- 主题结果标签（如 "本局抢位、压力上行、今日破局"）
- 折叠：「这一手为后面铺了什么」、「分享文案」
- 一行小字：「明天再来，会按新的流日重新出一题」
- 底部两个 CTA：看最近一月 / 进入人生主线

**模态：** 点 "?" 弹出半屏 sheet，解释题目怎么来、选项怎么读、做完之后会发生什么

### 3.4 人生游戏页（按时间范围）

主目标：让用户感觉在推进一条个人路线，不是阅读列表。

时间范围切换：今日 / 最近一月 / 最近一年 / 最近十年 / 一生主线（一生为默认）

- 顶部 header：范围 + 进度（"3 / 12 关"）+ 路线名（如"破局者 · 当前走到这里"）+ 一行"你最近的倾向：..."（用具体语言，不要"主动路线/稳健路线"裸字）
- 当前关卡主卡（最大视觉块）：阶段 kicker + 题目标题 + 情境 + 触发依据 meta
- 生活状态面板（默认折叠）：折叠时只显示一行 glance "压力 58 · 机会 64 · 关系 47"，展开后显示 6 维 + 一条状态提示
- 选择区：3 选项卡（结构同今日页）
- 反馈卡（结构同今日页，无明日提示）
- 下一关大按钮 / 完成态
- 行动日志（默认折叠）：简短列表，每条一行题目 + 一行所选动作
- 阶段地图（默认折叠）：3-4 个阶段卡，每个阶段一句策略

### 3.5 深度解读页

主目标：给好奇的人解释"为什么是这些题"。

- 顶部说明：这里解释为什么系统给出这些关卡
- 内容按专题分层显示：八字基础 / 紫微十二宫 / 格局 / 事业 / 财运 / 婚恋 / 健康 / 心性 / 依据
- 默认展开最相关 1-2 个专题，其余折叠
- 每个专题三层：**结论一句话** → **结构原因 2-4 条** → **展开依据（书籍/检索片段）**
- 依据折叠到最深层，不展示 AI prompt seed

### 3.6 命盘页（工具页）

给懂命理的人查盘面结构。手机端不强求十二宫盘面网格，默认列表。

- 基础信息
- 十二宫列表（可按主题筛选）
- 主星 / 辅星 / 四化
- 格局标签
- 当前重点宫位摘要（命宫、身宫、当前大运宫）

### 3.7 我的页

承载不适合放在主路径的信息。

- 当前出生信息
- 当前定位时间
- 当前路线 / 今日结果
- 重新生成、说明、免责声明

## 4. 希望的设计风格

### 4.1 视觉关键词

```
沉静 · 仪式感 · 新中式极简 · 暗色奢华 · 选择感 · 文化底蕴
```

气质要像一个**有质感的个人命运产品**，不是后台工具，也不是算命摊。神秘、克制、高级，带一点文化底蕴，但**现代而不复古**。

### 4.2 风格参考与最终方向

我们评估过四种候选风格，最终选定 **新中式极简 + 暗色奢华 混合** 作为本轮方向。四种参考如下，便于你理解我们要取哪一面、舍哪一面。

#### 候选 A：新中式极简（取它的"骨")

- 参考：Linear、Notion 的极简布局，但用中国传统元素点缀
- 配色：墨黑、宣纸白、朱砂红，金色作为点缀
- 字体：衬线中文字体（宋体质感）
- 感觉：高级、现代、有文化底蕴
- 适合人群：25-45 岁、有一定消费力、对传统文化感兴趣

#### 候选 B：国潮风（不取）

- 大胆使用传统纹样、山水插画，配色饱和度高（故宫红 / 青花蓝 / 翡翠绿）
- 适合 Z 世代，**与我们的目标用户和"克制"气质不符**

#### 候选 C：水墨禅意（不取）

- 大量留白、水墨渐变、深墨 / 淡灰 / 若有若无的青
- 神秘、沉静，**但太"玄学摆件"，不利于"个人命运游戏"的工具感**

#### 候选 D：暗色奢华（取它的"皮")

- 深色背景（近黑）+ 金色文字和线条
- 参考：Co–Star 占星 App
- 感觉：神秘、专业、高端、国际范

#### 最终方向：新中式极简 + 暗色奢华（混合）

**为什么混合：**

- 命理本身有神秘感，**暗色背景天然契合**
- 新中式让它**不显得土气、不显得算命摊**，有现代感
- **金色点缀**呼应"命理"的仪式感
- 在 App Store / 微信生态里**辨识度更高**，避免"又一个浅色卡片产品"的疲劳

**这意味着设计上：**

- 主背景以**近黑墨色**为主（不是纯黑，要带一点暖墨调），承载暗色奢华的"质感"
- 关键元素（标题、点缀线、状态标签）用**金色**，但严格克制使用面积，不要金色块
- 卡片表面用**宣纸白**或**极淡的暖灰**，让信息区从黑底里"浮"出来，承载新中式极简的"骨"
- **朱砂红**只用于"今日 / 当前 / 主按钮选中" 等极少强调点，绝不大面积铺
- 衬线中文字体用于**大标题和情境叙述**，正文 / 按钮 / 辅助说明用无衬线，避免老气
- 留白要足，间距要松，**不要堆砌纹样**——只用极轻的线条、微光、纸质纹理作为分隔

### 4.3 明确不要

- ❌ 后台管理系统风、表单工具感
- ❌ 资料库 / 多卡片堆叠
- ❌ 国风过度装饰（云纹、龙凤、印章、毛笔字标题）
- ❌ 大面积紫蓝渐变 / 大面积彩色块
- ❌ 廉价金色（俗气的镭射金、亮金块），金色只做线条与小面积点缀
- ❌ 粗糙的纯黑或漆黑，要带暖墨调的深色
- ❌ 大水墨晕染做主视觉（参见候选 C 不取的理由）
- ❌ 算命摊符号：八卦图、太极图、占星轮、塔罗卡
- ❌ 每个模块用不同高饱和色，破坏整体感
- ❌ 所有卡片都一样重

### 4.4 色彩方向（建议范围，待你定稿）

请围绕"暗色奢华 + 新中式极简"产出色板。**以下是建议范围，不是锁定值**，你可以依据视觉判断微调。

**主背景与表面（深暖墨基调）：**

```
建议范围：
page-bg:        #14110E ~ #1B1814   （近黑墨色，带极淡暖调，避免纯黑）
surface:        #1F1B16 ~ #26211B   （卡片底，从背景里浮出来一档）
surface-soft:   #2B2620 ~ #322C24   （次卡 / 折叠面板）
overlay:        rgba(255, 248, 232, 0.04 ~ 0.08)  （半透明高光层）
```

**主色（金色，承载点缀和仪式感）：**

```
建议范围：
primary-gold:        #C9A24A ~ #D6B25A   （主点缀、关键标题、当前态标识）
primary-gold-deep:   #8C6F2A ~ #A07C30   （hover/pressed、阴影层）
primary-gold-soft:   rgba(214, 178, 90, 0.14 ~ 0.22)  （次按钮底、tag 底）
```

**强调色（朱砂红，仅用于极少强调点）：**

```
建议范围：
accent-red:    #B23A2C ~ #C0432F   （主按钮选中、今日强提示）
accent-red-soft: rgba(192, 67, 47, 0.14)
```

**文字与线条（宣纸白系，避免纯白）：**

```
建议范围：
ink:        #F2E9D4 ~ #F7EFDB     （主文字，宣纸白，不刺眼）
ink-muted:  #B7AC95 ~ #C4B89F     （次要文字）
muted:      #7E7563 ~ #8B8270     （辅助说明、占位）
line:       rgba(242, 233, 212, 0.08 ~ 0.14)  （细分割线）
```

**状态色（克制，仅在反馈区使用）：**

```
positive:   #6B8C5A ~ #7A9B6A     （lifeState 正向变化标签）
warning:    #C99355 ~ #D49E5C     （提醒，不要刺眼红）
```

**配色使用比例参考：** 深暖墨基调 ≥ 70% · 宣纸白文字与浅卡 ≈ 20% · 金色点缀 ≈ 6-8% · 朱砂红 ≤ 2%

### 4.5 字体层级（手机端）

新中式极简的核心是**字体层次**。建议正文用无衬线（保证移动端可读），**大标题与情境叙述用衬线中文字体**（建立"读一段东西"的仪式感）。

**字体选择优先级：**

- 衬线中文（用于大标题、情境叙述、关卡题目）：`"Source Han Serif SC", "Noto Serif SC", "Songti SC", serif`
- 无衬线中文（用于正文、按钮、辅助说明、导航）：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif`
- 数字与英文（如 lifeState 数值、进度标识）：`"SF Pro Display", "Inter", system-ui, sans-serif`

**字号 / 行高 / 字重：**

```
页面大标题（衬线）：24-28px / line-height 1.2 / weight 500-600
关卡 / 情境标题（衬线）：20-22px / 1.3 / weight 500
卡片标题（无衬线）：17-18px / 1.3 / weight 600
正文（无衬线）：14-15px / 1.55 / weight 400
辅助说明（无衬线）：12-13px / 1.45 / weight 400
按钮（无衬线）：15px / 1.2 / weight 600
底部导航（无衬线）：11-12px / 1.1 / weight 500
金色数字 / 强调（数字字体）：14-22px，字重按场景调
```

**重要原则：** 衬线字体只用于"被阅读"的内容（题目、情境），不要用于按钮、tab、数据，避免老气。

### 4.6 布局与密度

- 页面左右 padding：16px
- 卡片间距：12-16px，大区块间距：22-28px（**比浅色版略松**，给暗色更多呼吸）
- 主卡 padding：20-22px，次卡 14-16px
- 单列为主，单手可点
- 主卡更强、次卡更轻，**不要一屏堆同权重卡片**
- 暗色版不要"厚边框包一切"——优先用**表面色差**和**金色细线**分隔
- 同一屏优先保证"可行动"，不是"看起来内容很多"
- 阴影克制：暗色版用阴影提升层级容易脏，**优先用"颜色更深的底"或"金色细线"代替阴影**

### 4.7 按钮

- 主按钮：高度 48px，圆角 14px，背景 `primary-gold` 或 `accent-red`（视场景），文字深墨色（不要白字压金色）
- 次按钮：高度 42px，圆角 12px，背景 `primary-gold-soft` 或 `surface-soft`，文字 `ink` 或 `primary-gold`，1px 金色细描边
- 文本按钮：无背景，`primary-gold` 文字，13-14px
- 选项卡（不是按钮，是大块可点区域）：默认 `surface` 底，选中 `surface` 底 + `primary-gold` 细描边 + 左侧金色短色条标识，被弱化态透明度降到 0.5

### 4.8 底部导航

- 高度 60px，背景 `surface` 与背景同色但带 4-6% 高光（玻璃感），顶部 1px 金色淡线
- 当前项：金色文字 + 顶部 18px 金色短线（线条比圆点更有质感）
- 未选项：`muted` 色文字
- 5 项均分宽度，标签最多 2 字，不换行、不横滑
- 图标可选：极简线性图标（描边而非填充），描边色随当前态金色 / 未选 muted

### 4.9 互动语气与文案

文案要**具体**，不要抽象主题词：

- ❌ "事业压力测试" / "关系边界课题" / "资源整合阶段"
- ✅ "今天有人把一个临时任务推给你，你要不要接？"
- ✅ "对方又在试探你的底线，你这次怎么回应？"

选项必须是**动作**，不是态度：

- ❌ "主动突破" / "稳健推进" / "修复关系"
- ✅ "直接接下任务，并争取把成果展示出来"
- ✅ "先确认边界，只接自己能负责的部分"

禁止绝对化预言："一定会发生 / 必有大灾 / 注定失败 / 投资必赚"。

推荐表达："更容易遇到 / 可能反复出现 / 适合提前处理 / 这个阶段更值得关注 / 如果选择主动推进，代价是……"

## 5. 希望产出

请围绕本 brief 给出以下内容（按优先级）：

1. **视觉方向定调稿**（必出）：一张 moodboard 或一段视觉语言描述，证明你接住了"新中式极简 + 暗色奢华"的混合气质。可附 2-3 张参考图说明取舍。
2. **色彩定稿**（必出）：在 §4.4 的建议范围内给出最终色值（page-bg / surface / surface-soft / primary-gold / primary-gold-deep / primary-gold-soft / accent-red / ink / ink-muted / muted / line / positive / warning），并给出一张包含全部色值的小色卡。
3. **字体定稿**（必出）：明确指定衬线中文字体与无衬线中文字体的具体方案（含 fallback），给出 6 个字号档位的实际渲染示例。
4. **核心三页高保真稿**（必出）：
   - 已生成态首页
   - 今日关卡选择前 + 选择后
   - 人生游戏页（一生主线视角，含当前关卡 + 反馈态）
5. **关键组件细节**（必出）：
   - 主卡 / 次卡 / 选项卡 / 反馈卡的尺寸、间距、字号、圆角、表面层级
   - 选项卡默认 / 选中 / 弱化（其余收起后）三态
   - 底部导航当前态 / 未选态
