# UI Reviewer

Use this skill after implementing UI.

## Goal

Review the page like a design lead, not like a compiler.

## Review dimensions

Always check these:

1. visual hierarchy
2. spacing
3. typography
4. card density
5. button clarity
6. color balance
7. first-screen clarity
8. mobile readability

## Output format

List findings in this order:

1. `最影响观感的问题`
2. `会影响使用的问题`
3. `可继续打磨的问题`

Then state:

- `建议先修哪两个`

## Review rules

- If everything looks same-weight, call it out.
- If whitespace wastes screen height, call it out.
- If cards are too tall for their information value, call it out.
- If buttons read like raw system controls, call it out.
- If a section feels like data dump rather than guided interface, call it out.

## For Mingli

Default aesthetic target:

- calm
- precise
- mysterious but not theatrical
- structured
- not admin-dashboard
- not generic form app

## Avoid weak approval

Do not say “looks good” unless it really holds up.
